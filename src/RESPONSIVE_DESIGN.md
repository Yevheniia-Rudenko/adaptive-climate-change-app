# Responsive Design Guide

## Overview
Приложение "Системы, которые формируют изменение климата" полностью адаптивно и оптимизировано для всех типов устройств, от мобильных телефонов до настольных компьютеров.

## Breakpoints (точки перелома)

Приложение использует стандартные Tailwind CSS breakpoints:

- **Mobile (базовый)**: < 640px - без префикса
- **sm (Small)**: ≥ 640px - планшеты в портретной ориентации
- **md (Medium)**: ≥ 768px - планшеты в альбомной ориентации
- **lg (Large)**: ≥ 1024px - ноутбуки и десктопы
- **xl (Extra Large)**: ≥ 1280px - большие десктопы

## Адаптивные компоненты

### 1. Header (Шапка)
- **Мобильные устройства**: 
  - Гамбургер-меню
  - Логотип без текста на экранах < 640px
  - Уменьшенный переключатель языков (scale-75)
  
- **Десктоп**: 
  - Горизонтальная навигация
  - Выпадающее меню модулей
  - Полный размер всех элементов

### 2. IntroPage (Вводная страница)
- **Изображения**: 
  - h-48 (мобильные) → h-64 (планшеты) → h-80 (десктопы)
  
- **Заголовки**: 
  - text-2xl → text-3xl → text-4xl → text-5xl
  
- **Отступы**: 
  - p-4 → p-6 → p-8 → p-10

### 3. FlexibleModulePage (Страница модуля)
- **Навигационные кнопки**: 
  - Мобильные: вертикальный стек (flex-col)
  - Десктоп: горизонтальный ряд (flex-row)
  - Кнопка "Complete" приоритетна на мобильных (order-1)

- **Заголовки модулей**:
  - text-xl → text-2xl → text-3xl → text-4xl

### 4. InteractiveDashboard (Интерактивная панель)
- **Метрики**: 
  - grid-cols-1 (мобильные) → grid-cols-2 (планшеты+)
  
- **Текст**: 
  - text-xs/sm для мобильных
  - text-sm/base для десктопа
  
- **Кнопка Reset**: 
  - w-full (мобильные) → w-auto (десктоп)
  
- **Контролы параметров**:
  - Вертикальная компоновка на мобильных
  - Горизонтальная на десктопе

### 5. ContentBlock (Блок контента)
- **Reflection блоки**:
  - min-h-[100px] → min-h-[120px]
  - p-4 → p-6 → p-8
  
- **Текстовые размеры**:
  - text-sm → text-base → text-lg

### 6. EndingPage (Финальная страница)
- **Иконки достижений**:
  - w-16 h-16 → w-20 h-20 → w-24 h-24
  
- **Заголовки**:
  - text-2xl → text-3xl → text-4xl → text-5xl

### 7. ProgressBar (Прогресс-бар)
- **Иконки**: size-20 → size-24
- **Переключатель языков**: scale-90 → scale-100
- **Отступы**: px-3 py-3 → px-4 py-4

### 8. Информационные страницы (About, Educators, Resources, Glossary)
- **Отступы контейнера**: 
  - px-4 py-8 → px-6 py-12
  
- **Внутренние отступы карточек**: 
  - p-6 → p-8 → p-12
  
- **Сетка (Educators)**:
  - grid-cols-1 → md:grid-cols-2

## Адаптивная типографика

Приложение использует responsive размеры шрифтов:
- **Мобильные**: меньшие размеры для экономии пространства
- **Планшеты**: средние размеры для комфортного чтения
- **Десктоп**: крупные размеры для лучшей читаемости

## Touch-friendly дизайн

### Минимальные размеры для тач-таргетов:
- Кнопки: минимум 44x44px (py-2.5, px-4)
- Иконки в кнопках: размер 18-20px
- Интерактивные элементы: gap-2 или gap-3

## RTL поддержка

Приложение поддерживает RTL (right-to-left) для арабского языка:
- Dropdown меню автоматически позиционируется справа для RTL
- Текст автоматически выравнивается по правому краю
- Отступы (margin) инвертируются (ml → mr)

## Лучшие практики

1. **Мобильный First**: Дизайн сначала для мобильных, затем расширение для больших экранов
2. **Гибкие единицы**: Использование относительных единиц (%, rem, vh/vw)
3. **Flexible Images**: Все изображения с `w-full h-auto`
4. **Min-width для контейнеров**: `max-w-4xl` для предотвращения слишком широких линий текста
5. **Гибкие сетки**: CSS Grid и Flexbox для адаптивных раскладок
6. **Touch-friendly**: Достаточно большие области для касания (минимум 44x44px)

## Производительность

- **Lazy loading** для изображений
- **Минимизация рендеринга**: Использование `flex-shrink-0` для предотвращения сжатия иконок
- **Transition классы**: Плавные анимации при изменении размеров
- **Viewport meta tag**: Корректное масштабирование на мобильных устройствах
